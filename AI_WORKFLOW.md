# AI_WORKFLOW.md — Evidencia de Uso de IA

El proyecto inicialmente empezo creando 2 agentes de copilot, uno enfocado en la arquitectura y planificacion se le dio acceso a las herramientas correctas para que solo ejecutara su trabajo y su funcionamiento es conocer el proyecto a detalle, sus reglas y estructura. Con el inicie cada desarrollo en todas las fases del proyecto.

Cree otro agente developer que se encarga de escribir codigo, crear archivos, probar funcionabilidades, etc.

Con estos 2 pidiendoles en el prompt que siempre sean precisos, cuidadoso y coherentes con todos los documento y el desarrollo construi esta aplicacion, en la ultima fase se utilizarian agentes en la nube, ya una vez todo el pipeline del deploy este configurado mi idea es utilizar agentes en la nube desde github para hacer el desarrollo mas comodo y directo sin necesidad de utilizar un IDE

En todo el proceso me toco guiar y hacer ajustes mediantes prompts hacia los distintos agentes para que mantuviera la mayor coherencia en el proyecto posible y haciendo ajustes en todo el desarrollo. Para el tema del frontend cree un diseno con stitch utilizando gemini pro 3.1 y lo deje como DESIGN_REFERENCE.html para que lo utilice como referencias, se crearon los technical docs y readme apartir de la IA tambien.

Se utilizaron los agentes de copilot utilizando principalmente el modelo Claude Opus 4.6
